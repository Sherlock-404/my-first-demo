## shell
- shell中大部分程序存放在/usr/bin/目录下
- GUI-图形化界面
- echo:用来在终端输出文本或变量的值
- 参数解析：类似python中的引号使用规则
![alt text](image.png)
- man(short for manual)：可以执行man程序来查看可以执行的命令
- tab:自动补全
- path:包含系统自带程序的路径
- which:可以查看某个命令的路径
- sed:流编辑器，可以用来处理文本文件，例如替换、删除、插入等操作
```
sed [选项] '操作' 文件名
/*eg:sed 's/Linux/linux/' test.txt
其中s表示替换，/Linux/表示要替换的字符串，/linux/表示替换后的字符串，test.txt表示要处理的文件名(默认只更改每行第一处)，这只是更改了输出结果，并没有更改原文件，如果要更改原文件，需要加上-i参数
*/
sed '2d' test.txt
/*表示删除test.txt文件的第2行

*/
sed '/Linux/d' test.txt
/*表示删除test.txt文件中包含Linux的行

*/
sed -n '2p' test.txt
/*表示打印test.txt文件的第2行

*/
sed '2i Hello' test.txt
/*表示在test.txt文件的第2行前插入Hello
如果要在第2行后插入Hello，则使用a命令；全部替换用c命令
*/

```
- grep:文本搜索工具，可以用来在文件中查找指定的字符串
- awk:文本处理工具，可以用来对文本文件进行格式化输出、统计等操作

- cut:文本处理工具，可以用来提取文本文件中的指定字段
- sort:文本处理工具，可以用来对文本文件进行排序
- uniq:文本处理工具，可以用来去除文本文件中的重复行
- find:文件搜索工具，可以用来在指定目录下查找指定的文件
- cat:文本处理工具，可以用来查看文本文件的内容
- tee:文本处理工具，可以用来将命令的输出同时写入文件和终端
- seq:可以用来生成一个指定范围的数字序列
- |:管道符，可以将前一个命令的输出作为后一个命令的输入
- \>:重定向符，可以将命令的输出写入文件 
- <:重定向符，可以将文件的内容作为命令的输入

## 正则表达式
正则表达式是一种用来描述字符串模式的工具，可以用来匹配、查找、替换等操作。常用的正则表达式符号包括：
- .:匹配任意一个字符
```

- *:匹配前一个字符零次或多次
- +:匹配前一个字符一次或多次
- ?:匹配前一个字符零次或一次
- ^:匹配字符串的开头
- $:匹配字符串的结尾
- []:匹配括号内的任意一个字符
- ():用于分组，可以将多个字符作为一个整体进行匹配
- |:表示或，可以匹配多个模式中的任意一个
- {}:用于指定匹配的次数，例如{n}表示匹配前一个字符n次，{n,}表示匹配前一个字符至少n次，{n,m}表示匹配前一个字符至少n次但不超过m次
